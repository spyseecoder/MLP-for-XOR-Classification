MLP for XOR Classification

An interactive exploration of a Multi-Layer Perceptron built from scratch to solve the classic, non-linearly separable XOR problem.

This section breaks down the architecture and mechanics of the neural network. The model was implemented in Python using only the NumPy library to provide a foundational understanding of the core concepts, from network structure to the training process.

Network Architecture

The MLP consists of three layers. This structure is essential for learning the non-linear patterns of the XOR problem, which a single-layer perceptron cannot solve.

Input Layer: 2 Neurons
↓
Hidden Layer: 4 Neurons (ReLU Activation)
↓
Output Layer: 1 Neuron (Sigmoid Activation)

Activation & Loss

The choice of activation and loss functions is critical. ReLU was used in the hidden layer for its efficiency, while Sigmoid in the output layer constrains the prediction to a probability between 0 and 1. Binary Cross-Entropy loss measures the performance, which is ideal for this binary classification task.

Forward Propagation

During the forward pass, input data flows through the network. At each layer, a weighted sum of inputs is calculated, and an activation function is applied. This process continues until the final output layer produces a prediction.

Backpropagation & Training

Training is an iterative process of refining the model's weights and biases. The backpropagation algorithm calculates the gradient of the loss function. These gradients are then used in gradient descent to update the network's parameters, incrementally minimizing the error and improving the model's accuracy over 1600 epochs.

This interactive diagram visualizes the core of the project: the MLP's decision boundary. The colored regions show how the trained model separates the data space to classify the XOR inputs. This demonstrates the network's ability to learn a complex, non-linear function.
